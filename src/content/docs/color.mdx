## Overview 

The color space used by SOL is [okLCH](https://developer.mozilla.org/en-US/docs/Web/CSS/color_value/oklch). Why okLCH? It's much easier to predict color contrast compared to using other color spaces like RGB or HSL. 

## Color Schemes

SOL supports light and dark mode by default. All colors are defined using the `light-dark()` notation. However, sometimes a project may only require a light scheme. Feel free to redefine the `--color-scheme` property.

```css
:root {
  --color-scheme: light dark;
}
```

## Color Palettes

SOL uses a concept of palettes, ~~brazenly borrowed~~ shamelessly stolen from [Open Props](https://open-props.style/) instead of commonly defined color swatches typically found in libraries like TailwindCSS. Thereby, theming is much simpler with less CSS.

More info:  
[Open Props v2 Color Guide](https://opv2-beta.netlify.app/color/)  
[Open Props oklch Palettes Beta](https://nerdy.dev/open-props-oklch-palettes-beta)

SOL provides 4 color palettes: 

```css
--color-primary-{1-16}  
/* (Brand color scale) */ 

--color-secondary-{1-16}  
/* (Secondary color scale, optional) */ 
  
--color-gray-{1-16}  
/* (Neutral color scale) */

--color-utility-{1-16}  
/* (Utility colors such as success, warning, error) */  
```

### Primary Color Palette

[View primary color swatches](#color-palette-primary)

Override the default primary hue with any pre-defined `--hue-*` values, or use a number: 

```css
--primary-hue: var(--hue-*)
/* 
--hue-{red | pink | purple | violet | indigo | blue | cyan | teal | green | lime | yellow | orange} 
*/
```

[See all hue options](#hue-options)

### Secondary Color Palette

[View secondary color swatches](#color-palette-secondary)

Similar to setting the primary hue, to override the default secondary hue, use any pre-defined `--hue-*` value, or use a number: 

```css
--secondary-hue: var(--hue-*) 
/* 
--hue-{red | pink | purple | violet | indigo | blue | cyan | teal | green | lime | yellow | orange} 
*/
```

### Gray Color Palette

[View gray color swatches](#color-palette-gray)

By default, the gray palette tone is influenced by the `--primary-hue` by `0.1` of chroma which gives the neutral scheme a subtle hint of primary color. However, any of these settings can be changed.

The default gray palette settings: 

```css
--gray-hue: var(--primary-hue)
--gray-hue-rotation: 0
--gray-chroma: 0.1
```

### Utility Colors 

Utility colors serve a specific purpose; not meant for branding. For example, a red color might be used to indicate an alert. 

To use a specific utility color, define the `--utility-hue` first. Then specify a number from the `--color-utility-*` scale.

#### CSS Example: 

```css
mark {
  --utility-hue: var(--hue-yellow);
  background: var(--color-utility-4);
}
```
